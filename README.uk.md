# Autosound Tuning Skill

🇬🇧 [English](README.md) · 🇩🇪 [Deutsch](README.de.md) · 🇵🇱 [Polski](README.pl.md) · 🇺🇦 **Українська**

**Одним рядком:** скіл для Claude, що веде тебе до чистого, прозорого, збалансованого звуку у *твоїй* машині — приносить усе ремесло під твій конкретний сетап, читає твої заміри з REW і допомагає обрати кожну зміну.

- 📊 **Працює з REW** через API — витягує заміри, заливає EQ-фільтри назад
- 🎯 **Знає ремесло** — цільові криві, практики тюнінгу, покроковий процес
- 🎧 **Тест-треки** — що слухати і на якому треку (описи, не аудіо)
- 🚗 **Вчить твій сетап** — накопичує знання про авто й обладнання, лише за твоєю згодою

**Для кого — Seeker (шукач).** Кожен, хто будує власний звук і ще вчиться ремеслу (не лише досвідчений майстер). Ти даєш вуха й машину — скіл дає мапу.

**Навіщо.** Тюнінг — це лавина: забагато методик, параметрів і порад, щоб тримати в голові, і легко зануритися в одну деталь і згубити цілісну картину. Скіл — твій навігатор: тримає знання, вказує на ті кілька змін, що справді важать, і тримає в полі зору компроміс між сценою та тональним балансом. Твоє вухо — кінцевий суддя.

Він охоплює повний тюн — від нового проєкту (опитування обладнання + цілей, вибір цільової кривої, перевірка інсталяції) через кросовери, часове вирівнювання, фазу, поканальний і сумарний EQ та побудову сцени, аж до войсингу на смак — керований циклом рев'ю Генератор ↔ Критик ↔ Арбітр. Це **метод, а не машина**: заміри авто чи стан DSP тут не зберігаються (вони лишаються у твоєму проєкті), тож він працює на **будь-якому авто / будь-якому DSP**.

> Збудований і випробуваний у боях на VW Passat B8 / Helix DSP Ultra S (перемога на змаганнях AYA); специфіка авто/DSP винесена в окремий профіль + бібліотеку `knowledge/`, тож новий проєкт — це просто «обери вхідні дані».

## Що тут є

```
autosound-tuning-skill/        плагін Claude Code
└── skills/autosound-tuning/    скіл
    ├── SKILL.md        точка входу — мапа процесу, життєвий цикл сесії, ролі
    ├── references/     документи на вимогу (фази, діагностика, EQ, фільтри, сцена,
    │                   тест-треки, REW API, Helix, метод рев'ю, інтейк …)
    ├── knowledge/      накопичені профілі авто й DSP (cars/, dsp/)
    └── scripts/        приклад інструментарію каналу Критика (опційно)
```

Один скіл — незалежний метод рев'ю (Критик/Радник/Арбітр, анти-анкоринг) вкладено як `references/core/review-loop.md`.

## Початок роботи — уперше в Claude Code?

Цей скіл працює в **Claude Code** — інструменті в терміналі. Тюнінг ітеративний (заміряти → аналізувати → змінити → переміряти) над текстовими/числовими даними, тож термінал + git — парсинг, скриптинг, повна історія змін — пасують значно краще за вебдодаток.

**1. Встанови Claude Code** (macOS / zsh; потрібні Node.js 18+ і платний план Claude — Pro / Max / Team / Enterprise):
```bash
node --version                                   # Node.js 18 або новіший
mkdir -p ~/.npm-global && npm config set prefix '~/.npm-global'
echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
npm install -g @anthropic-ai/claude-code
claude --version                                 # перевірка
```

**2. Створи теку проєкту і запусти:**
```bash
mkdir ~/car-audio-setup && cd ~/car-audio-setup
claude                                            # перший запуск відкриє браузер для входу
```

**3. Почни просто.** Порожня тека, перший експорт REW кинутий як є, один markdown-лог нотаток (лише дописуй) — а підтеки нехай з'являються, коли реально знадобляться.

**4. Твоє перше повідомлення = «крок 0».** Опиши систему: авто + розкладка динаміків (скільки смуг, чи є саб?), модель DSP, вимірювальне обладнання, і чи вже маєш заміри. Далі скіл веде сам — і **спершу питає твою бажану робочу мову** (EN · UK · DE · PL), щоб і розмова, і файли проєкту виходили нею.

## Вимоги

- **Claude Code** (або claude.ai зі скілами).

```

Оновлення згодом: `/plugin update autosound-tuning`.

*(Більше до вподоби ручний checkout? Склонуй репо і зроби symlink **внутрішньої** теки `skills/autosound-tuning` у `~/.claude/skills/`. ⚠️ Не клонуй увесь репозиторій *у* `~/.claude/skills/autosound-tuning/` — `SKILL.md` тоді ляже на рівень глибше, і Claude повідомить `Unknown skill`.)*

## Як поділитися досвідом

Скіл **вчиться з кожного тюну — збираючи фідбек прямо в терміналі, поки ти працюєш, а не через анкету.** На завершенні (коли ти задоволений звуком) він питає, що допомогло, що було не так, і про будь-який каверз DSP/авто — тоді, **за твоєю явною згодою**, пропонує поділитися *узагальнюваними* уроками (щоб розвивати спільний метод + бібліотеку `knowledge/`).

Він фіксує **лише метод + класи обладнання** — поведінку салону, клас DSP/обладнання, які техніки спрацювали. **Ніколи персональних даних, ніколи повних замірів;** ти бачиш точно, що надсилається, і погоджуєшся пункт за пунктом. Підтверджені уроки інтегруються в скіл з атрибуцією; суперечливі поради зберігаються як *варіанти*, а не видаляються. *(Більше до вподоби GitHub? Відкрий [field-feedback issue](../../issues/new?template=field-feedback.md) — те саме правило.)*
