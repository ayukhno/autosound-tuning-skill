**Одним рядком:** скіл для Claude, що веде тебе до чистого, прозорого, збалансованого звуку у *твоїй* машині — приносить усе ремесло під твій конкретний сетап, читає твої заміри з REW і допомагає обрати кожну зміну.

- 📊 **Працює з REW** через API — витягує заміри, заливає EQ-фільтри назад
- 🎯 **Знає ремесло** — цільові криві, практики тюнінгу, покроковий процес
- 🎧 **Тест-треки** — що слухати і на якому треку (описи, не аудіо)
- 🚗 **Вчить твій сетап** — накопичує знання про авто й обладнання, лише за твоєю згодою

## Початок роботи

Цей скіл працює як плагін для **Claude Code** (офіційного термінального агента від Anthropic).

### 1. Швидке встановлення

Усередині активної сесії Claude Code запустіть ці команди **по черзі, по одній** (не копіюйте і не вставляйте їх разом):

```bash
/plugin marketplace add ayukhno/autosound-tuning-skill
```

```bash
/plugin install autosound-tuning
```

```bash
/reload-plugins
```

*Тоді почніть тюн, сказавши:* **"налаштуй нове авто з нуля"** (або англійською: *"tune a new car from scratch"*).

### 2. Повне налаштування та FAQ

Потрібна допомога зі встановленням Claude Code, запуском на **Windows**, конфігурацією **Gemini Critic** або вибором вимірювального мікрофона?

## Для кого і навіщо

* **Для кого:** Для тих, хто будує звук в авто і вчиться цьому ремеслу. Це твій екзоскелет, який за допомогою твого слуху та дій (там, де немає прямого програмного інтерфейсу) керує знаннями й досвідом, налаштовуючи автозвук твоєї мрії.
* **Навіщо:** Тюнінг — це лавина: забагато методик, параметрів і порад, щоб тримати в голові, і легко зануритися в одну деталь і згубити цілісну картину. Скіл — твій навігатор: тримає знання, вказує на ті кілька змін, що справді важать, і тримає в полі зору компроміс між сценою та тональним балансом. Твоє вухо — кінцевий суддя.

Він охоплює повний тюн — від нового проєкту через кросовери, часове вирівнювання, фазу, поканальний і сумарний EQ, побудову сцени й до фінального войсингу на смак — керований циклом рев'ю **Генератор ↔ Критик ↔ Арбітр**.

## Як поділитися досвідом

Скіл вчиться з кожного тюну — збираючи фідбек прямо в терміналі, поки ти працюєш, а не через анкету. На завершенні (коли ти задоволений звуком) він питає, що допомогло, що було не так, і про будь-який каверз DSP/авто — тоді, **за твоєю явною згодою**, пропонує поділитися *узагальнюваними* уроками (щоб розвивати спільний метод + бібліотеку `knowledge/`).

Він фіксує **лише метод + класи обладнання** — поведінку салону, клас DSP/обладнання, які техніки спрацювали. **Ніколи персональних даних, ніколи повних замірів;** ти бачиш точно, що надсилається, і погоджуєшся пункт за пунктом. Підтверджені уроки інтегруються в скіл з атрибуцією.
