# Autosound Tuning Skill

🇬🇧 [English](README.md) · 🇩🇪 [Deutsch](README.de.md) · 🇵🇱 [Polski](README.pl.md) · 🇺🇦 **Українська**

**Одним рядком:** скіл для Claude, що допомагає налаштовувати автозвук — читає твої заміри з REW, аналізує їх, радить налаштування і вміє залити EQ назад у REW.

- 📊 **Працює з REW** через API — витягує заміри, заливає EQ-фільтри назад
- 🎯 **Знає ремесло** — цільові криві, практики тюнінгу і вбудований покроковий процес налаштування
- 🎧 **Тест-треки** — що слухати і на якому треку (описи, не аудіо)
- 🚗 **Вчить твій сетап** — накопичує знання про авто й обладнання під кутом тюнінгу (лише за твоєю згодою)

Скіл для [Claude](https://claude.com/claude-code) для налаштування DSP автозвуку з [REW](https://www.roomeqwizard.com/) — від нового проєкту з нуля (опитування обладнання + цілей, вибір цільової кривої, верифікація інсталяції) через кросовери, часове вирівнювання, фазу, поканальний і сумарний EQ, побудову сцени/локалізації — аж до войсингу під смак клієнта. Він кодує виміряний **метод** (цикл рев'ю Генератор ↔ Критик ↔ Арбітр) плюс зростаючу бібліотеку вистражданих діагностичних технік.

Скіл — це **метод, а не машина.** Заміри жодного авто чи стан DSP тут не зберігаються — вони лишаються у власному проєкті тюнера. Цей репозиторій містить багаторазовий процес, тож він працює на **будь-якому авто / будь-якому DSP**.

> Збудований і випробуваний у боях на VW Passat B8 / Helix DSP Ultra S (перемога на змаганнях AYA), але специфіка авто/DSP винесена в окремий профіль + бібліотеку `knowledge/`, тож новий проєкт — це просто «обери вхідні дані».

## Що тут є

```
skills/
├── autosound-tuning/      головний скіл
│   ├── SKILL.md           точка входу (мапа процесу, життєвий цикл сесії, ролі)
│   ├── references/        документи на вимогу: фази, діагностика, EQ-патерни,
│   │                      типи фільтрів, сцена/глибина, змагання, тест-треки,
│   │                      войсинг на слух, REW API, специфіка Helix, інтейк, фідбек
│   ├── knowledge/         накопичені профілі авто й DSP (cars/, dsp/)
│   └── scripts/           приклад реалізації каналу Критика (опційний інструментарій рига)
└── review-loop/           сусідній скіл: оркестрація незалежного рев'ю (vendor-agnostic)
```

Ці два скіли — **пара**: `autosound-tuning` посилається на `review-loop` за протоколом рев'ю. Встанови обидва.

## Початок роботи — уперше в Claude Code / терміналі?

Цей скіл працює в **Claude Code** — інструменті в терміналі. Якщо досі ти користувався лише десктоп/веб-чатом, ось одноразове налаштування. *(Інші мови: [English](README.md) · [Deutsch](README.de.md) · [Polski](README.pl.md).)*

**Чому Claude Code (а не вебдодаток / Cowork):** тюнінг ітеративний — заміряти → аналізувати → змінити → переміряти — над текстовими/числовими даними (CSV/текстові експорти REW, конфіги DSP, логи ітерацій). Термінал + git дають парсинг, скриптинг і повну історію змін. Cowork орієнтований на операції з файлами в інших застосунках — тут це надмірно.

**1. Встанови Claude Code** (macOS / zsh; потрібні Node.js 18+ і платний план Claude — Pro / Max / Team / Enterprise; безкоштовний план не спрацює):
```bash
node --version                                   # Node.js 18 або новіший
mkdir -p ~/.npm-global && npm config set prefix '~/.npm-global'
echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
npm install -g @anthropic-ai/claude-code

```

**2. Створи теку проєкту і запусти:**
```bash

claude                                            # перший запуск відкриє браузер для входу
```

**3. Не переструктуровуй наперед.** Почни з порожньої теки. Кинь туди свій перший експорт REW (`.txt`/`.csv`) як є, веди один markdown-лог нотаток (лише дописуй), і дозволь текам (`measurements/`, `configs/`, `eq-log/`) з'явитися тоді, коли реально знадобляться.

**4. Твоє перше повідомлення = «крок 0».** Опиши систему одним повідомленням: авто + розкладка динаміків (скільки смуг, чи є саб?), модель DSP (Helix / Audison / інший), вимірювальне обладнання (мік, інтерфейс), і чи в тебе вже є заміри, чи стартуєш від заліза. Далі скіл веде сам — і **спершу питає твою бажану робочу мову** (English / рідна, якщо підтримується: EN · UK · DE · PL), щоб і розмова, і файли проєкту виходили нею. Тоді: інтейк → перевірка тракту → калібрування міка → перший свип → …

## Вимоги

- **Claude Code** (або claude.ai зі скілами).
- **REW** з увімкненим API-сервером (`localhost:4735`), калібрований вимірювальний мік.
- **ПЗ твого DSP** і спосіб залити в нього EQ (ідеально — імпорт файлу; для 30+ DSP без імпорту див. [REW-EQ-CopyPaste-Assistant](https://github.com/IvanBakhmutov/REW-EQ-CopyPaste-Assistant)).
- **Опційно:** друга AI-модель як «Критик» (будь-який вендор — ролі vendor-agnostic). Без неї роль рев'юера грає людина; процес усе одно тримається.

## Встановлення

**Найпростіше — хай Claude встановить за тебе.** Відкрий Claude Code і просто попроси, наприклад:

> *«Склонуй https://github.com/ayukhno/autosound-tuning-skill у тимчасову теку, тоді скопіюй дві **внутрішні** теки скілів — `skills/autosound-tuning` і `skills/review-loop` — у мою теку рівня користувача `~/.claude/skills/`, щоб `SKILL.md` кожного скіла опинився прямо в `~/.claude/skills/<name>/SKILL.md`. Не клонуй увесь репозиторій у теку скілів.»*

Claude розмістить **обидва** скіли (вони — пара) там, де обереш:

- **Рівень користувача — `~/.claude/skills/`** *(рекомендовано)*: доступний у **кожній** теці, де відкриваєш Claude Code. Обери це, якщо налаштовуватимеш більш ніж одне авто або хочеш мати скіл завжди під рукою.
- **Рівень проєкту — `<твій-проєкт>/.claude/skills/`**: лише в одному проєкті. Обери це, щоб репозиторій одного авто був самодостатнім.

> ⚠️ **Типова помилка встановлення:** не роби `git clone` репозиторію *у* `~/.claude/skills/autosound-tuning/`. Це вкладе `SKILL.md` на рівень глибше (`…/autosound-tuning/skills/autosound-tuning/SKILL.md`), і Claude Code повідомить **`Unknown skill: autosound-tuning`** (або взагалі не запропонує скіл). Завжди копіюй **внутрішні** теки `skills/*`, щоб `SKILL.md` лежав *прямо* в теці скіла.

**Або вручну:**
```bash
git clone https://github.com/ayukhno/autosound-tuning-skill.git
# рівень користувача (доступно скрізь) — копіюй ВНУТРІШНІ теки скілів:
cp -R autosound-tuning-skill/skills/autosound-tuning ~/.claude/skills/
cp -R autosound-tuning-skill/skills/review-loop      ~/.claude/skills/
# …або рівень проєкту: cp -R ті самі дві теки у  your-project/.claude/skills/

# перевірка — кожен рядок має вивести шлях (SKILL.md лежить ПРЯМО в теці скіла):
ls ~/.claude/skills/autosound-tuning/SKILL.md
ls ~/.claude/skills/review-loop/SKILL.md
```

**Після встановлення запусти НОВУ сесію Claude Code** — скіли завантажуються на старті.

**Усунення проблем — `Unknown skill` / скіл не тригериться:** майже завжди це вкладеність вище. Якщо `ls ~/.claude/skills/autosound-tuning/SKILL.md` падає, а `…/autosound-tuning/skills/autosound-tuning/SKILL.md` існує — ти склонував увесь репозиторій у теку скіла. Виправ, тоді перезапусти Claude Code:
```bash
mv ~/.claude/skills/autosound-tuning ~/.claude/skills/autosound-tuning-repo
cp -R ~/.claude/skills/autosound-tuning-repo/skills/autosound-tuning ~/.claude/skills/
cp -R ~/.claude/skills/autosound-tuning-repo/skills/review-loop      ~/.claude/skills/
rm -rf ~/.claude/skills/autosound-tuning-repo
```

Тоді відкрий Claude Code у своєму проєкті і скажи, напр. *«налаштуй нове авто з нуля»* — скіл почне з **інтейку** (`references/project-intake.md`): швидкий старт, опитування обладнання + цілей, вибір цільової кривої (без дефолту — обирається з тобою), верифікація інсталяції і генерація файлів проєкту. *(Спершу він питає твою бажану робочу мову — увесь локальний проєкт можна вести своєю мовою; англійська тут лише внутрішній метод скіла.)*

## Як поділитися досвідом

Скіл **вчиться з кожного тюну — і збирає цей фідбек прямо в терміналі, поки ти працюєш, а не через анкету для заповнення.** На завершенні (Фаза 7) він проводить короткий закриваючий ритуал: питає, що реально допомогло, що було не так, і про будь-який каверз DSP/авто, на який ти натрапив — тоді, **за твоєю явною згодою**, пропонує поділитися *узагальнюваними* уроками.

**Для чого фінальне опитування і що воно збирає:** щоб розвивати спільний метод + бібліотеку `knowledge/`. Воно фіксує **лише метод + класи обладнання** — поведінку кузова/салону, DSP/процесор і клас обладнання, і які техніки спрацювали. **Ніколи персональних даних, ніколи повних замірів.** Ти бачиш точно, що буде надіслано, і погоджуєшся пункт за пунктом (або ні).

Підтверджені уроки фолдяться у скіл і профілі `knowledge/` авто/DSP (з атрибуцією). Поради, що суперечать наявним висновкам, зберігаються як *варіанти*, а не видаляються — інший салон може зробити їх слушними.

*(Більше до вподоби GitHub? Можеш так само відкрити [field-feedback issue](../../issues/new?template=field-feedback.md) — те саме правило: лише метод/класи обладнання.)*
