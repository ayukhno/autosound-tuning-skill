# Autosound Tuning Skill

🇬🇧 [English](README.md) · 🇩🇪 [Deutsch](README.de.md) · 🇵🇱 [Polski](README.pl.md) · 🇺🇦 **Українська**

**Одним рядком:** скіл для Claude, що допомагає налаштовувати автозвук — читає твої заміри з REW, аналізує їх, радить налаштування і вміє залити EQ назад у REW.

- 📊 **Працює з REW** через API — витягує заміри, заливає EQ-фільтри назад
- 🎯 **Знає ремесло** — цільові криві, практики тюнінгу, покроковий процес
- 🎧 **Тест-треки** — що слухати і на якому треку (описи, не аудіо)
- 🚗 **Вчить твій сетап** — накопичує знання про авто й обладнання, лише за твоєю згодою

Він охоплює повний тюн — від нового проєкту (опитування обладнання + цілей, вибір цільової кривої, перевірка інсталяції) через кросовери, часове вирівнювання, фазу, поканальний і сумарний EQ та побудову сцени, аж до войсингу під смак — керований циклом рев'ю Генератор ↔ Критик ↔ Арбітр. Це **метод, а не машина**: заміри авто чи стан DSP тут не зберігаються (вони лишаються у твоєму проєкті), тож він працює на **будь-якому авто / будь-якому DSP**.

> Збудований і випробуваний у боях на VW Passat B8 / Helix DSP Ultra S (перемога на змаганнях AYA); специфіка авто/DSP винесена в окремий профіль + бібліотеку `knowledge/`, тож новий проєкт — це просто «обери вхідні дані».

## Що тут є

```
skills/
├── autosound-tuning/   головний скіл
│   ├── SKILL.md        точка входу — мапа процесу, життєвий цикл сесії, ролі
│   ├── references/     документи на вимогу (фази, діагностика, EQ, фільтри, сцена,
│   │                   тест-треки, REW API, Helix, інтейк, фідбек …)
│   ├── knowledge/      накопичені профілі авто й DSP (cars/, dsp/)
│   └── scripts/        приклад інструментарію каналу Критика (опційно)
└── review-loop/        сусідній скіл: оркестрація незалежного рев'ю
```

Ці два — **пара**: `autosound-tuning` використовує `review-loop` за протоколом рев'ю. Встанови обидва.

## Початок роботи — уперше в Claude Code?

Цей скіл працює в **Claude Code** — інструменті в терміналі. Тюнінг ітеративний (заміряти → аналізувати → змінити → переміряти) над текстовими/числовими даними, тож термінал + git — парсинг, скриптинг, повна історія змін — пасують значно краще за вебдодаток.

**1. Встанови Claude Code** (macOS / zsh; потрібні Node.js 18+ і платний план Claude — Pro / Max / Team / Enterprise):
```bash
node --version                                   # Node.js 18 або новіший
mkdir -p ~/.npm-global && npm config set prefix '~/.npm-global'
echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc
npm install -g @anthropic-ai/claude-code
claude --version                                 # перевірка
```

**2. Створи теку проєкту і запусти:**
```bash
mkdir ~/car-audio-setup && cd ~/car-audio-setup
claude                                            # перший запуск відкриє браузер для входу
```

**3. Почни просто.** Порожня тека, перший експорт REW кинутий як є, один markdown-лог нотаток (лише дописуй) — а підтеки нехай з'являться, коли реально знадобляться.

**4. Твоє перше повідомлення = «крок 0».** Опиши систему: авто + розкладка динаміків (скільки смуг, чи є саб?), модель DSP, вимірювальне обладнання, і чи вже маєш заміри. Далі скіл веде сам — і **спершу питає твою бажану робочу мову** (EN · UK · DE · PL), щоб і розмова, і файли проєкту виходили нею.

## Вимоги

- **Claude Code** (або claude.ai зі скілами).
- **REW** з увімкненим API-сервером (`localhost:4735`) і калібрований мік.
- **ПЗ твого DSP** + спосіб залити EQ (ідеально — імпорт файлу; для 30+ DSP без нього див. [REW-EQ-CopyPaste-Assistant](https://github.com/IvanBakhmutov/REW-EQ-CopyPaste-Assistant)).
- **Опційно:** друга AI-модель як «Критик» (будь-який вендор). Без неї рев'ю робить людина; процес усе одно тримається.

## Встановлення

**Найпростіше — хай Claude зробить це.** Відкрий Claude Code і скажи:

> *«Склонуй https://github.com/ayukhno/autosound-tuning-skill у тимчасову теку, тоді скопіюй дві **внутрішні** теки `skills/autosound-tuning` і `skills/review-loop` у `~/.claude/skills/`, щоб `SKILL.md` кожного опинився прямо в `~/.claude/skills/<name>/SKILL.md`. Не клонуй увесь репозиторій у теку скілів.»*

Встанови **обидва** (вони — пара) на **рівні користувача — `~/.claude/skills/`** (доступно скрізь, рекомендовано) або **рівні проєкту — `<проєкт>/.claude/skills/`** (щоб репозиторій одного авто був самодостатнім).

> ⚠️ **Єдина помилка, якої слід уникати:** не роби `git clone` репозиторію *у* `~/.claude/skills/autosound-tuning/`. Це вкладе `SKILL.md` на рівень глибше, і Claude повідомить **`Unknown skill`**. Завжди копіюй **внутрішні** теки `skills/*`, щоб `SKILL.md` лежав *прямо* в теці скіла.

**Вручну:**
```bash
git clone https://github.com/ayukhno/autosound-tuning-skill.git
cp -R autosound-tuning-skill/skills/autosound-tuning ~/.claude/skills/
cp -R autosound-tuning-skill/skills/review-loop      ~/.claude/skills/
# перевірка — кожен має вивести шлях (SKILL.md прямо в теці скіла):
ls ~/.claude/skills/autosound-tuning/SKILL.md
ls ~/.claude/skills/review-loop/SKILL.md
```

**Тоді запусти НОВУ сесію Claude Code** (скіли завантажуються на старті) і скажи, напр. *«налаштуй нове авто з нуля»* — скіл почне з **інтейку**: швидкий старт, опитування обладнання + цілей, вибір цільової кривої (обирається з тобою), перевірка інсталяції, генерація файлів проєкту. *(Якщо `Unknown skill` або скіл не тригериться — майже завжди це вкладеність вище; перекопіюй внутрішні теки `skills/*` і перезапусти.)*

## Як поділитися досвідом

Скіл **вчиться з кожного тюну — збираючи фідбек прямо в терміналі, поки ти працюєш, а не через анкету.** На завершенні (Фаза 7) він питає, що допомогло, що було не так, і про будь-який каверз DSP/авто — тоді, **за твоєю явною згодою**, пропонує поділитися *узагальнюваними* уроками (щоб розвивати спільний метод + бібліотеку `knowledge/`).

Він фіксує **лише метод + класи обладнання** — поведінку салону, клас DSP/обладнання, які техніки спрацювали. **Ніколи персональних даних, ніколи повних замірів;** ти бачиш точно, що надсилається, і погоджуєшся пункт за пунктом. Підтверджені уроки фолдяться у скіл з атрибуцією; суперечливі поради зберігаються як *варіанти*, а не видаляються. *(Більше до вподоби GitHub? Відкрий [field-feedback issue](../../issues/new?template=field-feedback.md) — те саме правило.)*
