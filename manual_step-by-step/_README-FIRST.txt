1. ЧИМ ЧИТАТИ ФАЙЛИ .MD (MARKDOWN), ЩОБ ЦЕ БУЛО КРАСИВО ТА ЧИТАБЕЛЬНО?
-------------------------------------------------------------------------------
Усі шаблони кроків та інструкції в цій папці написані у форматі Markdown (.md). 
Вони містять таблиці, списки та діаграми. Якщо відкрити їх як звичайний текст, 
вони будуть захаращені символами #, * та |.

Щоб читати їх у зручному, відрендереному вигляді (як гарну веб-сторінку):

* ВАРІАНТ А (Рекомендований для ПК): VS Code (Visual Studio Code)
  Відкрийте будь-який .md файл у VS Code та натисніть комбінацію клавіш:
  - На macOS:  Cmd + Shift + V
  - На Windows: Ctrl + Shift + V
  Це відкриє інтерактивне вікно попереднього перегляду (Markdown Preview).

* ВАРІАНТ Б (Безкоштовний супер-інструмент): Obsidian
  Obsidian (obsidian.md) — це безкоштовний додаток для нотаток для Windows, 
  macOS, iOS та Android. Просто перетягніть папку "manual_step-by-step" 
  в Obsidian як нове сховище (Vault). Він ідеально відобразить усі таблиці, 
  посилання та навіть інтерактивні схеми-діаграми.

* ВАРІАНТ В (Швидкий без софту): GitHub / GitLab
  Оскільки цей проєкт живе на GitHub, ви можете просто відкрити вашу сторінку 
  репозиторію в браузері. GitHub автоматично рендерить .md файли у чудовому, 
  зручному для читання стилі.

* ВАРІАНТ Г (У браузері): Розширення для Chrome/Firefox
  Встановіть безкоштовне розширення "Markdown Viewer" у свій браузер. Після цього
  ви зможете просто перетягувати будь-який .md файл прямо в робоче вікно браузера,
  і він відображатиметься як чистий, красивий документ.

2. ЗОЛОТЕ ПРАВИЛО: СУВОРИЙ STATELESS-РЕЖИМ (ЗАХИСТ ВІД ГАЛЮЦИНАЦІЙ ШІ)
-------------------------------------------------------------------------------
ВЕБ-ЧАТИ (ChatGPT, Claude, Gemini) МАЮТЬ ОБМЕЖЕНУ ПАМ'ЯТЬ (Context Window).
Коли розмова триває більше 1-2 годин, модель починає "дрейфувати": плутає затримки,
забуває частоти зрізів, помиляється в математиці та робить суперечливі поради.

* ПРАВИЛО:* Кожен новий крок налаштування — це абсолютно НОВИЙ, ЧИСТИЙ чат-сеанс!
* Алгоритм:*
  1. Оновили свій файл "autosound_context.md" на диску.
  2. Відкрили абсолютно НОВИЙ чат із ШІ.
  3. Скопіювали відповідний промпт кроку (наприклад, step_2_tonal_balance_eq.md).
  4. Надіслали його разом із вмістом вашого "autosound_context.md".
  5. Отримали розрахунки, внесли їх у DSP.
  6. Закрили/видалили цей чат та відкрили новий для наступного кроку.

Це на 100% захистить вас від математичних помилок ШІ та зекономить купу часу!

3. ТЕХНІКА БЕЗПЕКИ: НЕ СПАЛІТЬ ДИНАМІКИ!
-------------------------------------------------------------------------------
* Твітери (ВЧ) та середньочастотники (СЧ) є дуже тендітними фізично.
* НІКОЛИ не запускайте вимірювальний свип (Sweep) у REW на ВЧ/СЧ без активного
  захисного фільтра високих частот (HPF) в DSP!
* Безпечні стартові фільтри для замірів:
  - Для ВЧ (твітери): HPF не нижче 1000-2000 Гц (зазвичай не менше 1.1 Fs динаміка), крутизна 24 дБ/окт (наприклад, LR4).
  - Для СЧ: HPF не нижче 100-300 Гц (зазвичай не менше 1.1 Fs динаміка), крутизна 24 дБ/окт (наприклад, LR4).
* Завжди робіть перший тестовий замір на дуже низькій гучності!

4. ВАШ ЛОКАЛЬНИЙ ПАСПОРТ СИСТЕМИ (autosound_context.md) — ХТО ЙОГО ЗАПОВНЮЄ?
-------------------------------------------------------------------------------
Файл "autosound_context.md" — це єдине джерело правди про вашу систему, на якому 
тримається весь конвеєр. Процес його заповнення дуже простий:

А. ЯК СТВОРИТИ ЦЕЙ ФАЙЛ НА СТАРТІ? (Два шляхи):
   * ШЛЯХ 1 (ШІ робить сам): Ви копіюєте промпт із "step_0_intake_and_setup.md" 
     у новий чат. ШІ проведе з вами коротке інтерв'ю (питатиме по 2-3 питання 
     за раз про авто, динаміки, підсилювачі). Ви відповідаєте звичайними словами,
     а наприкінці ШІ сам згенерує і видасть вам готовий структурований текст. 
     Вам залишиться лише скопіювати його та зберегти як "autosound_context.md".
   * ШЛЯХ 2 (Швидкий старт руками): Якщо ви не хочете грати в інтерв'ю — просто 
     візьміть наш готовий шаблон "autosound_context_template.md", відкрийте в 
     Блокноті, замініть текстові заготовки у дужках (наприклад, [Модель DSP]) 
     на свої реальні дані і збережіть як "autosound_context.md".

Б. ЯК ВІН ОНОВЛЮЄТЬСЯ ПІД ЧАС ТЮНІНГУ? (Принцип копіювання результатів):
   Коли ви виконуєте Крок 1, 2 чи 3, ШІ вираховуватиме для вас нові значення 
   (затримки, зрізи кросовера, смуги еквалайзера чи кути фаз). 
   Після внесення цих налаштувань у свій DSP (через софт вашого процесора), ви 
   просто копіюєте ці фінальні таблиці чи цифри з вікна чату і вставляєте їх у 
   відповідні розділи вашого локального файлу "autosound_context.md".

Це забирає 30 секунд часу, але ваш "сейв гри" завжди актуальний! Коли ви підете 
на наступний крок, новий чистий чат отримає оновлений файл і миттєво зрозуміє 
поточний стан вашої системи без жодних додаткових розпитувань.
